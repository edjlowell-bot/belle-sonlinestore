## 🚀 Deploy to Production (Free — 15 minutes)

### Step 1: Get the code on GitHub

1. Go to **github.com** → Sign in or create free account
2. Click **New repository** → Name it `belles-enterprise`
3. Set to **Private** (important — protects your business data)
4. Click **Create repository**
5. Upload all these files by dragging them into the GitHub interface  
   OR use GitHub Desktop (easier): https://desktop.github.com

---

### Step 2: Deploy to Vercel

1. Go to **vercel.com** → Sign in with your GitHub account
2. Click **Add New Project**
3. Import your `belles-enterprise` repository
4. Vercel auto-detects Vite — no config needed
5. Before clicking Deploy, go to **Environment Variables** and add:

```
VITE_SUPABASE_URL      → https://yourproject.supabase.co
VITE_SUPABASE_KEY      → eyJhbGci... (your anon key)
VITE_GEMINI_KEY        → AIzaSy... (optional, for AI invoices)
```

6. Click **Deploy**
7. Your app is live at: `https://belles-enterprise.vercel.app`

> **Every time you update a file on GitHub, Vercel auto-redeploys in ~30 seconds.**

### Step 3: Set up Supabase (if not done yet)

#### Required Tables

**transactions**
```sql
create table public.transactions (
  id          bigint generated by default as identity primary key,
  created_at  timestamptz default now(),
  date        date not null,
  miner_name  text not null,
  sales       numeric(12,2) default 0,
  paid        numeric(12,2) default 0,
  fulfillment text default 'Pickup',
  payment_mode text default 'GCash',
  status      text default 'Pending'
);
```

**expenses**
```sql
create table public.expenses (
  id           bigint generated by default as identity primary key,
  created_at   timestamptz default now(),
  date         date not null,
  category     text not null,
  amount       numeric(12,2) default 0,
  payment_mode text default 'GCash',
  notes        text
);
```

**investors**
```sql
create table public.investors (
  id        bigint generated by default as identity primary key,
  name      text not null,
  equity    numeric(5,2) default 0,
  capital   numeric(12,2) default 0,
  withdrawn numeric(12,2) default 0
);
```

#### Row Level Security (RLS) — Highly Recommended
```sql
-- Enable RLS on all tables
alter table public.transactions enable row level security;
alter table public.expenses     enable row level security;
alter table public.investors    enable row level security;

-- Allow anon reads (your app authenticates via API key)
create policy "Allow anon read" on public.transactions for select using (true);
create policy "Allow anon write" on public.transactions for all using (true);

create policy "Allow anon read" on public.expenses for select using (true);
create policy "Allow anon write" on public.expenses for all using (true);

create policy "Allow anon read" on public.investors for select using (true);
```

## 💻 Local Development (StackBlitz or VS Code)

### For StackBlitz:
1. Import from GitHub: `stackblitz.com/github/YOUR_USERNAME/belles-enterprise`
2. Create `.env` file with your Supabase keys
3. Run: `npm install && npm run dev`

### For VS Code:
```bash
npm install
cp .env.example .env.local
# Edit .env.local with your credentials
npm run dev
```

## 🔐 Security Notes

- **Never commit `.env` or `.env.local`** — these are in `.gitignore`
- Supabase anon key is safe to use client-side (it's designed for this)
- For production, enable Supabase RLS policies (SQL above)
- Credentials are stored in `sessionStorage` — auto-cleared on tab close
- Set your Supabase project to **not** allow signups (Auth → Settings → Disable email signups) since you're not using Supabase Auth
